# 淘宝用户行为数据集 — 数据探查与异常分析

## 数据概述

本数据集来自阿里巴巴提供的淘宝用户购物行为数据集（UserBehavior），包含 2017 年 11 月 25 日至 2017 年 12 月 3 日之间，约一百万随机用户的所有行为（点击、购买、加购、收藏），共计 **100,095,182** 条记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | 整型 | 序列化后的用户 ID |
| item_id | 整型 | 序列化后的商品 ID |
| category_id | 整型 | 商品所属类目 ID |
| behavior | 字符串 | pv / buy / cart / fav |
| timestamp | 整型 | Unix 时间戳 |

数据清洗后保留 2017-11-25 ~ 2017-12-03 共 9 天的有效数据，过滤掉了约 5 万条脏数据（时间戳超出范围）。

## 一、行为分布

| 行为 | 记录数 | 占比 |
|------|--------|------|
| pv（浏览） | 89,660,671 | 89.58% |
| cart（加购） | 5,530,446 | 5.53% |
| fav（收藏） | 2,888,258 | 2.89% |
| buy（购买） | 2,015,807 | 2.01% |

行为数量排序为 **pv > cart > fav > buy**，与实际生产环境一致。

### 整体转化率（行为笔数）

```
CVR = buy笔数 / pv笔数 = 2,015,807 / 89,660,671 = 2.25%
```

这个数值符合电商平台的典型水平（2-5%）。

### 整体转化率（独立用户数）

```
pv_uv      = 984,105   （有浏览行为的独立用户）
interest_uv = 987,882   （有浏览/加购/收藏行为的独立用户）
buy_uv     = 672,404   （有购买行为的独立用户）

CVR = buy_uv / interest_uv = 68.07%
```

**68% 看起来异常偏高？** 不是。这里统计的是"在整个 9 天内曾经购买过的用户占曾经有过任何行为的用户的比例"。由于数据集筛选的是活跃用户（大部分用户有 10+ 条行为记录），68% 的用户在 9 天内至少买过一次东西是合理的。

## 二、核心异常：46.4% 的购买在当天没有浏览记录

这是本数据集最关键的发现。

```sql
-- 按 (date, user_id, item_id) 统计当天是否有 pv 和 buy
SELECT
    COUNT(*) AS buy_cnt,
    SUM(has_pv) AS click_cnt
FROM (
    SELECT date, user_id, item_id,
        MAX(IF(behavior='pv', 1, 0)) AS has_pv,
        MAX(IF(behavior='buy', 1, 0)) AS has_buy
    FROM user_behavior
    GROUP BY date, user_id, item_id
) t
WHERE has_buy = 1;
```

**结果：**

| 指标 | 数值 |
|------|------|
| 当天有 buy 的 (user, item) 对数 | 1,966,120 |
| 其中当天有 pv 的 | 1,053,586（53.6%） |
| **当天无 pv 的** | **912,534（46.4%）** |

**超过一半的购买行为，在当天找不到对应的浏览记录。** 这意味着用户在数据期前就已经浏览/加购了商品，在数据期内直接从购物车或收藏夹下单。

## 三、商品转化率异常分析

### 3.1 商品 CVR 分布

对满足最低浏览量（interest_uv ≥ 10）的商品统计 CVR 分布：

| CVR 区间 | 商品数 | 占比 |
|----------|--------|------|
| < 5% | 826,300 | 86.4% |
| 5-10% | 126,178 | 13.2% |
| 10-50% | 68,500 | 7.2% |
| 50-100% | 458 | 0.05% |
| 100-200% | 51 | 0.005% |
| 200-500% | 15 | 0.002% |
| > 500% | 1 | 0.0001% |

**绝大多数商品（86.4%）的 CVR 低于 5%，符合电商行业规律。** 但有 67 个商品的 CVR 超过 100%，其中 14 个超过 200%。

### 3.2 极端 CVR 商品（> 200%）

| 商品ID | pv_uv | cart_uv | fav_uv | buy_uv | interest_uv | CVR |
|--------|-------|---------|--------|--------|-------------|-----|
| 1881953 | 18 | 0 | 0 | 127 | 18 | 705.56% |
| 10497 | 23 | 0 | 0 | 72 | 23 | 313.04% |
| 4244177 | 35 | 0 | 0 | 97 | 35 | 277.14% |
| 3557522 | 105 | 0 | 6 | 293 | 111 | 263.96% |
| 1411566 | 39 | 0 | 2 | 108 | 41 | 263.41% |
| 2628872 | 18 | 0 | 0 | 47 | 18 | 261.11% |
| 1777337 | 26 | 0 | 0 | 65 | 26 | 250.00% |
| 331334 | 65 | 0 | 1 | 157 | 66 | 237.88% |
| 1927740 | 16 | 0 | 0 | 36 | 16 | 225.00% |
| 3236112 | 13 | 0 | 0 | 29 | 13 | 223.08% |

### 3.3 CVR > 100% 的根因验证

对上述极端商品逐个验证：**购买用户中，有多少人在数据期内浏览过该商品？**

| 商品ID | buy_uv | pv_uv | 无pv的buy用户 | 占比 |
|--------|--------|-------|--------------|------|
| 1881953 | 127 | 18 | 123 | **97%** |
| 10497 | 72 | 23 | 68 | **94%** |
| 4244177 | 97 | 35 | 88 | **91%** |
| 3557522 | 293 | 105 | 266 | **91%** |
| 1411566 | 108 | 39 | 92 | **85%** |

**结论：CVR 超过 100% 的商品，85%-97% 的购买者在数据期内没有浏览过该商品。**

这些用户通过以下渠道购买，但不产生 pv 记录：
1. **购物车直接结算** — 数据期前加购，数据期内直接下单
2. **收藏夹直接购买** — 数据期前收藏，数据期内直接下单
3. **搜索结果页直接购买** — 不进入详情页，不产生 pv
4. **推荐流直接购买** — 信息流中直接操作

### 3.4 类目维度同样存在极端值

| 类目ID | pv_uv | buy_uv | CVR |
|--------|-------|--------|-----|
| 2923285 | 7 | 761 | 10,871% |
| 2464463 | 3 | 102 | 3,400% |
| 1781126 | 28 | 866 | 3,093% |
| 1712001 | 46 | 433 | 941% |
| 1404020 | 511 | 1,041 | 203% |

这些类目的共同特征：**pv_uv 极低，但 buy_uv 很高。** 大部分购买来自数据期前的加购/收藏行为。

## 四、转化漏斗分析

### 4.1 严格递进漏斗

每一步只统计完成上一步的用户：

| 阶段 | 用户数 | 占比 |
|------|--------|------|
| 浏览（pv） | 984,105 | 100% |
| 浏览 + 加购/收藏 | 855,498 | 86.9% |
| 浏览 + 加购/收藏 + 购买 | 600,278 | 61.0% |

**86.9% 的浏览用户有加购/收藏行为** — 这个比例偏高，原因是数据集筛选的是活跃用户，且 9 天时间窗口内大部分用户都会产生加购/收藏。

**61.0% 的浏览+加购/收藏用户完成了购买** — 同样因为活跃用户筛选和跨期效应。

### 4.2 为什么漏斗比例偏高？

真实电商平台的漏斗通常是：
- 浏览 → 加购/收藏：5-15%
- 加购/收藏 → 购买：30-60%
- 整体：2-5%

我们的数据偏高原因：
1. **数据集筛选了活跃用户** — 大部分用户有 10+ 条行为记录
2. **跨期效应** — 数据期前的加购/收藏在数据期内完成购买
3. **9 天窗口内用户行为重复** — 同一用户多次浏览同一商品

## 五、用户行为特征

### 5.1 用户行为数分布

| 行为数区间 | 用户数 |
|-----------|--------|
| < 10 | 18,638 |
| 10-50 | 308,391 |
| 50-100 | 286,564 |
| 100-500 | 371,474 |
| ≥ 500 | 2,924 |

**98.1% 的用户有 10 条以上行为记录**，说明数据集经过了用户活跃度过滤，只保留了行为较多的用户。

### 5.2 跨期用户分析

| 日期 | 首次出现的用户数 |
|------|----------------|
| 2017-11-25（第一天） | 706,641 |
| 2017-12-03（最后一天） | 1 |

第一天有 70 万用户首次出现，说明大部分用户在数据期前就已经是淘宝用户。这些用户带着历史购物车和收藏夹进入数据期，导致购买行为与浏览行为脱节。

## 六、异常值总结与解释

| 现象 | 数据表现 | 根因 |
|------|---------|------|
| 商品 CVR > 100% | 67 个商品，最高 705% | 85-97% 的购买者在数据期内无浏览记录 |
| 类目 CVR > 1000% | 5 个类目，最高 10,871% | 同上，类目级别的聚合放大了效应 |
| 整体 CVR = 68% | 用户级转化率偏高 | 活跃用户筛选 + 跨期效应 |
| 漏斗 86.9% → 61% | 远高于行业水平 | 活跃用户 + 跨期 + 9天窗口重复行为 |
| 46.4% 购买无当日浏览 | 接近一半的购买脱节 | 购物车/收藏夹的跨期持久性 |
| 行为级 CVR = 2.25% | 符合行业水平 | 这是唯一可靠的转化率指标 |

## 七、对分析结论的影响

### 7.1 哪些指标是可信的？

| 指标 | 可信度 | 原因 |
|------|--------|------|
| 行为分布（pv 90%, cart 5%, fav 3%, buy 2%） | ✅ 可信 | 与行业一致 |
| 行为级 CVR（2.25%） | ✅ 可信 | 符合行业水平 |
| 每日 UV/PV 趋势 | ✅ 可信 | 不受跨期影响 |
| 用户活跃时段分布 | ✅ 可信 | 不受跨期影响 |
| 用户级 CVR（68%） | ⚠️ 偏高 | 活跃用户筛选 |
| 商品 CVR | ❌ 严重偏高 | 分母不完整 |
| 漏斗转化率 | ⚠️ 偏高 | 跨期 + 活跃用户 |

### 7.2 报告中应如何说明？

在展示商品 CVR 和漏斗数据时，应注明：

> 由于数据集仅记录详情页浏览（pv），不包含搜索结果页、推荐流等渠道的曝光行为，且数据期仅 9 天，大量用户的历史加购/收藏行为未被纳入统计，导致商品维度的 CVR 指标分母不完整，数值系统性偏高。行为级整体 CVR（2.25%）更能反映真实的转化水平。
